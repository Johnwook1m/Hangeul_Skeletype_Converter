## 베타 테스트 개요

베타 테스트는 플러그인의 기능, 사용성, 안정성을 실제 사용자 환경에서 검증하는 중요한 단계입니다. 체계적인 베타 테스트를 통해 정식 릴리스 전에 문제점을 발견하고 개선할 수 있습니다.

**베타 테스트 핵심 사항:**
- **목적**: 기능 검증, 사용성 평가, 안정성 확인, 실제 사용 사례 검증
- **테스터**: 한글 폰트 디자이너, 타이포그래피 연구자, 다양한 경험 수준의 사용자
- **기간**: 권장 2-4주
- **인원**: 최소 3-5명, 이상적 5-10명

---

## 베타 테스트의 목적

베타 테스트를 통해 다음을 확인하고 개선합니다:

### 1. 기능 검증
- 다양한 한글 글리프에서 중심선 추출이 정확하게 작동하는지
- 다양한 폰트 스타일(세리프, 산세리프, 손글씨 등)에서의 호환성
- 복잡한 글리프 구조에서의 안정성

### 2. 사용성 평가
- 설치 과정이 명확하고 쉬운지
- 사용 방법이 직관적인지
- 오류 메시지가 이해하기 쉬운지
- 다이얼로그 흐름이 자연스러운지

### 3. 안정성 확인
- 다양한 시스템 환경에서의 작동 여부 (macOS 버전, Glyphs 3 버전)
- 메모리 누수나 성능 문제
- 예외 상황 처리 (빈 글리프, 복잡한 글리프 등)

### 4. 실제 사용 사례 검증
- 실제 작업 환경에서의 유용성
- 워크플로우에 자연스럽게 통합되는지
- 개선이 필요한 부분 파악

## 플러그인의 사용 목적 및 사용 사례

이 플러그인은 다음 목적으로 사용될 수 있습니다:

### 1. 스켈레톤 타입 폰트 제작
- 한글 폰트의 중심선을 추출하여 스켈레톤 타입 기반 폰트 제작
- 기존 폰트를 스켈레톤 타입으로 변환

### 2. 폰트 분석 및 연구
- 기존 폰트의 구조 분석
- 중심선을 통한 글리프 형태 연구

### 3. 폰트 디자인 워크플로우
- 초기 스케치에서 중심선 추출
- 스켈레톤을 기반으로 한 폰트 확장

### 4. 교육 및 학습
- 한글 타이포그래피 교육 자료 제작
- 글리프 구조 이해를 위한 시각화

## 베타 테스터 선정 기준

다음 조건을 만족하는 테스터를 선정하는 것을 권장합니다:

### 필수 조건
- ✅ **Glyphs 3 사용 경험**: Glyphs 3를 사용해본 경험이 있어야 함
- ✅ **macOS 사용자**: 플러그인이 macOS 전용이므로 필수
- ✅ **터미널 사용 가능**: Homebrew 설치 및 명령어 실행 가능

### 우선 선정 대상

#### 1. 한글 폰트 디자이너
- 실제 한글 폰트 제작 경험이 있는 디자이너
- 다양한 스타일의 한글 글리프를 테스트할 수 있음

#### 2. 타이포그래피 연구자/학생
- 폰트 구조에 대한 이해가 있는 연구자
- 분석적 관점에서 피드백 제공 가능

#### 3. 다양한 경험 수준의 사용자
- **초보자**: 사용성 평가
- **중급자**: 기능 검증
- **고급자**: 고급 기능 및 워크플로우 평가

#### 4. 다양한 시스템 환경
- Apple Silicon (M1/M2) Mac 사용자
- Intel Mac 사용자
- 다양한 macOS 버전 사용자

### 테스터 수
- **최소 3-5명**: 다양한 관점에서 피드백 수집
- **이상적 5-10명**: 더 넓은 범위의 테스트 및 검증

## 베타 테스터를 위한 설치

### 중요 사항

**플러그인이 작동하려면 다음 3가지가 모두 필요합니다:**

1. **플러그인 파일**: `HanguelSkeletypeConverter.glyphsPlugin`
2. **ImageMagick**: 시스템에 설치되어 있어야 함
3. **Autotrace**: 시스템에 설치되어 있어야 함

**⚠️ 플러그인 파일만 전달하면 작동하지 않습니다!** 베타 테스터가 ImageMagick과 Autotrace를 별도로 설치해야 합니다.

### 전달할 파일

베타 테스터에게 전달할 파일:
```
HanguelSkeletypeConverter.glyphsPlugin  (약 168KB)
```

베타 테스터가 직접 설치해야 할 것:
- ImageMagick (Homebrew로 설치)
- Autotrace (Homebrew로 설치)

### 베타 테스터 필수 요구사항

#### 1. ImageMagick

터미널에서 설치:
```bash
brew install imagemagick
```

설치 확인:
```bash
which magick || which convert
```

#### 2. Autotrace

터미널에서 설치:
```bash
brew install autotrace
```

설치 확인:
```bash
which autotrace
```

#### 3. Homebrew가 없는 경우

먼저 Homebrew 설치:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

자세한 설치 방법은 [PLUGIN_GUIDE.md](PLUGIN_GUIDE.md)의 "설치 및 사용" 섹션을 참고하세요.

- 체계적인 데이터 수집 가능

### 방법 3: GitHub Issues (대규모 또는 공개)
- 각 테스터가 Issue로 피드백 제출
- 버그 추적 및 관리 용이
- 토론 및 후속 질문 가능
